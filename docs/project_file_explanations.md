# Пояснение по файлам проекта

Ниже перечислены основные файлы учебного проекта. После каждого пути коротко объясняется, зачем этот файл нужен и что в нем происходит.

## `app/Models/User.php`
Модель пользователя. Она описывает поля пользователя, связи с вопросами и ответами, а также простой метод `totalScore()`, который считает суммарный рейтинг пользователя по его вопросам и ответам.

## `app/Models/Question.php`
Модель вопроса. В ней есть связи с автором, ответами, тегами и голосами. Метод `score()` считает сумму лайков и дизлайков, а `userVote()` помогает понять, как проголосовал текущий пользователь.

## `app/Models/Answer.php`
Модель ответа. Работает почти так же, как модель вопроса, только относится к конкретному вопросу. Здесь тоже есть методы для рейтинга и просмотра голоса текущего пользователя.

## `app/Models/Tag.php`
Модель тега. Нужна для хранения тематик вопросов, например `супы`, `мясо`, `десерты`. Через связь `belongsToMany` один тег может относиться ко многим вопросам.

## `app/Models/QuestionVote.php`
Модель голоса за вопрос. Хранит, кто и как проголосовал за вопрос. Значение `1` означает лайк, а `-1` означает дизлайк.

## `app/Models/AnswerVote.php`
Модель голоса за ответ. Работает по такому же принципу, как и голосование за вопрос, но для таблицы ответов.

## `app/Http/Controllers/HomeController.php`
Контроллер главной страницы. Он получает список вопросов из базы данных, добавляет к ним автора, теги, рейтинг и количество ответов. Здесь же находится логика поиска по заголовку, тексту вопроса, тексту ответа и тегам.

## `app/Http/Controllers/QuestionController.php`
Контроллер вопросов. Метод `create()` показывает форму создания вопроса, `store()` сохраняет новый вопрос, а `show()` открывает страницу одного вопроса вместе с ответами. Внутри контроллера есть отдельный метод, который разбирает строку тегов и привязывает их к вопросу.

## `app/Http/Controllers/AnswerController.php`
Контроллер ответов. Он принимает текст ответа, проверяет его через простую валидацию и сохраняет ответ к выбранному вопросу.

## `app/Http/Controllers/AuthController.php`
Контроллер авторизации. Отвечает за регистрацию, вход и выход из системы. Здесь используется стандартный механизм Laravel: проверка данных, хеширование пароля и создание сессии пользователя.

## `app/Http/Controllers/ProfileController.php`
Контроллер профиля. Он загружает пользователя, его вопросы и ответы, а затем передает эти данные в шаблон профиля для вывода статистики и активности.

## `app/Http/Controllers/QuestionVoteController.php`
Контроллер голосования за вопросы. Он принимает значение лайка или дизлайка, сохраняет его и умеет убирать голос, если пользователь нажал ту же кнопку повторно.

## `app/Http/Controllers/AnswerVoteController.php`
Контроллер голосования за ответы. Логика аналогична голосованию за вопросы, только работает с ответами.

## `routes/web.php`
Главный файл маршрутов. Здесь описано, какие URL есть у сайта и какой контроллер на них отвечает. В этом файле также видно, какие страницы доступны только авторизованным пользователям.

## `database/migrations/2026_04_13_180000_add_bio_to_users_table.php`
Миграция добавляет поле `bio` в таблицу пользователей. Оно нужно для короткого описания пользователя на странице профиля.

## `database/migrations/2026_04_13_180100_create_questions_table.php`
Создает таблицу `questions`. В ней хранятся заголовок вопроса, описание, автор и даты создания и обновления.

## `database/migrations/2026_04_13_180200_create_answers_table.php`
Создает таблицу `answers`. В ней сохраняются ответы, их авторы и связь с конкретным вопросом.

## `database/migrations/2026_04_13_180300_create_tags_table.php`
Создает таблицу тегов. Теги помогают группировать вопросы по темам.

## `database/migrations/2026_04_13_180400_create_question_tag_table.php`
Создает промежуточную таблицу между вопросами и тегами. Она нужна, потому что у одного вопроса может быть несколько тегов, и один тег может использоваться у разных вопросов.

## `database/migrations/2026_04_13_180500_create_question_votes_table.php`
Создает таблицу голосов за вопросы. В таблице хранится один голос конкретного пользователя для конкретного вопроса.

## `database/migrations/2026_04_13_180600_create_answer_votes_table.php`
Создает таблицу голосов за ответы. Логика такая же, как у таблицы голосов за вопросы.

## `database/seeders/DatabaseSeeder.php`
Заполняет базу начальными данными. В проект добавлен тестовый пользователь и базовый набор кулинарных тегов, чтобы сайт не выглядел пустым после первого запуска.

## `resources/views/layouts/app.blade.php`
Общий шаблон сайта. В нем находятся шапка, поиск, меню, вывод сообщений об успехе и ошибках. Остальные страницы подставляют в этот шаблон свое содержимое.

## `resources/views/home/index.blade.php`
Шаблон главной страницы. Показывает список вопросов, популярные теги, результаты поиска и переключение между страницами.

## `resources/views/questions/create.blade.php`
Форма создания вопроса. Пользователь вводит заголовок, описание и теги, после чего данные отправляются в контроллер.

## `resources/views/questions/show.blade.php`
Страница одного вопроса. Здесь выводятся текст вопроса, теги, рейтинг, список ответов и форма для добавления нового ответа.

## `resources/views/auth/login.blade.php`
Страница входа. На ней расположена простая форма с email и паролем.

## `resources/views/auth/register.blade.php`
Страница регистрации. Пользователь вводит имя, email, пароль и короткое описание о себе.

## `resources/views/profile/show.blade.php`
Страница профиля пользователя. Здесь показаны данные автора, общая статистика, его вопросы и его ответы.

## `public/css/app.css`
Один общий файл стилей. В нем описано оформление сайта: шапка, кнопки, карточки вопросов, формы, профиль и адаптация под маленький экран.

## `docs/run_mysql.md`
Короткая инструкция по запуску проекта на MySQL. Этот файл пригодится при настройке окружения перед демонстрацией работы преподавателю.

## `docs/coursework.md`
Полный текст курсовой работы в формате Markdown. Этот файл нужен для чтения текста и как исходник для генерации Word-документа.

## `scripts/generate_coursework_docx.ps1`
PowerShell-скрипт, который собирает `.docx` без сторонних библиотек. Он создает структуру Word-документа, применяет оформление, добавляет содержание, основную часть и нумерацию страниц.
