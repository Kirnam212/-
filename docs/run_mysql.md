# Как переключить проект на MySQL

Сейчас в локальном `.env` проекта включен `sqlite`, потому что это самый простой вариант для быстрого старта. Сам код проекта написан на обычном Eloquent и без привязки к конкретной СУБД, поэтому его можно без проблем использовать с MySQL.

Для запуска на MySQL достаточно изменить параметры в `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=cookoverflow
DB_USERNAME=root
DB_PASSWORD=
```

После этого нужно:

1. Создать пустую базу данных `cookoverflow`.
2. Выполнить миграции командой `php artisan migrate`.
3. При необходимости заполнить стартовые теги командой `php artisan db:seed`.

Почему проект будет работать и в MySQL:

- используются стандартные миграции Laravel;
- связи построены через внешние ключи `foreignId`;
- голосование и теги хранятся в обычных таблицах без сложных расширений;
- валидация и запросы написаны через стандартный Eloquent.
